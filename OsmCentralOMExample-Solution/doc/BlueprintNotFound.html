Blueprint not available

Blueprints are created as part of the project build process. A Blueprint is created (or recreated) when the entity is problem free and a full or incremental build is run which includes the entity.

A Blueprint may be unavailable for the following reasons:

  • Problem markers exist on the related entity preventing creation of the Blueprint.
  • The project build is disabled and entity has not been created.
  • Incremental builder may consider the entity as up-to-date, but the generated blueprint is not present.
  • The build is currently running and the blueprint has not yet been produced.

To resolve the issue:

  • If the project is building, wait for it to complete. The in-progress build may resolve the issue.
  • Check if problem markers exist on this entity. Error markers will prevent the Blueprint from generating. Resolve outstanding errors and save.
  • Enable Build Automatically on the Project menu, or Build Project manually.
  • If the build is enabled and the entity has no error markers, Clean the project to force a build of the entity.
